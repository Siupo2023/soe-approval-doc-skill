# 国企内部请示行文风格指南

## 一、总体要求

### 基本风格
- **正式庄重**：使用正式书面语，避免口语化
- **客观理性**：基于事实，用数据和事实说话
- **逻辑严密**：论证充分，因果关系清晰
- **简洁精准**：言简意赅，避免冗余
- **规范得体**：符合公文格式和层级文化

### 语言特点
- ✅ 使用完整句式，主谓宾齐全
- ✅ 多用书面语词汇
- ✅ 恰当使用公文常用语
- ❌ 避免网络流行语
- ❌ 避免过于随意的表达

---

## 二、标准结构

### 完整文档结构

```markdown
# 关于XXXX的请示

**公司领导：** / **集团公司：**

[开篇段落：简明扼要说明目的、背景、政策依据，1-3句话]

## 一、基本情况/背景与意义
[说明当前情况、存在的问题、为什么需要关注]

## 二、必要性与可行性分析
[论证为什么必须做、为什么能做、预期效果]

## 三、具体方案/主要内容/工作建议
[详细的行动计划、措施、步骤]

## 四、风险分析与防范措施
[识别潜在风险，提出应对措施]

## 请示事项
现将有关事项请示如下：
一、XXXX
二、XXXX
三、XXXX

妥否，请批示。

**附件：**
1. XXXX方案
2. XXXX清单

[部门名称]
2025年05月25日
```

## 三、各部分写作要点

### 标题
**格式**：`关于XXXX的请示`

**要求**：
- 准确概括核心事项
- 简洁明了，一般不超过20字
- 不用标点符号（除书名号）
- 示例：
  - ✅ 关于采购办公设备的请示
  - ✅ 关于修订财务管理制度的请示
  - ❌ 关于申请采购一些电脑设备的请示（太口语）

### 称谓
**格式**：顶格写，后加冒号

**常见称谓**：
- 公司领导：
- 集团公司：
- 董事会：
- 总经理：

### 开篇段落
**作用**：开门见山，说明为什么写这份请示

**要素**：
- 目的/事项
- 背景/依据（政策、会议、上级要求等）
- 简要意义

**示例**：
```
为提升部门工作效率，根据公司2025年度信息化建设规划，
拟采购一批办公自动化设备。现将有关事项请示如下。
```

### 主体部分

#### 一、基本情况/背景与意义

**写作要点**：
- 说明现状
- 指出问题/需求
- 分析重要性

**常用句式**：
- "近年来，..."
- "当前，我部..."
- "随着......的发展，..."
- "......存在以下问题："
- "......具有重要意义"

#### 二、必要性与可行性分析

**写作要点**：
- **必要性**：为什么必须做
- **可行性**：为什么能做
- **紧迫性**：为什么现在做

**常用句式**：
- "此举......"
- "......是......的必然要求"
- "......具备......条件"
- "......已具备......基础"

#### 三、具体方案/主要内容

**写作要点**：
- 详细具体
- 条理清晰
- 可操作性强

**常用句式**：
- "拟采取以下措施："
- "具体包括以下几个方面："
- "一是......；二是......；三是......"

#### 四、风险分析与防范措施

**写作要点**：
- 客观识别风险
- 提出具体应对措施
- 体现审慎态度

**常用句式**：
- "可能存在......风险"
- "针对上述风险，拟采取以下措施："
- "......防范于未然"

### 请示事项

**格式**：
- 使用"现将有关事项请示如下："引出
- 分条列出，用"一、""二、""三、"
- 明确具体，便于审批

**示例**：
```
现将有关事项请示如下：
一、同意采购办公设备15台，预算金额XX万元；
二、同意采用公开招标方式采购；
三、请财务部安排资金预算。
```

### 结尾

**标准表达**：
- 妥否，请批示。
- 特此请示，请批复。
- 以上请示，如无不妥，请批示。

**选择**：
- 请求批准 → "妥否，请批示。"
- 请求指示 → "特此请示，请批复。"

### 附件

**格式**：
```
附件：
1. XXXX方案
2. XXXX清单
3. XXXX报告
```

**注意**：附件名称要与实际文件一致

### 落款

**格式**：
- 右对齐
- 部门名称（全称）
- 日期（中文大写：YYYY年MM月DD日）

**示例**：
```
                                   综合管理部
                                   2025年05月25日
```

## 四、语言风格规范

### 常用词汇

#### 动词
- 拟、拟议、拟定
- 批准、审批、核准
- 制定、修订、完善
- 实施、执行、落实
- 遵照、依照、按照

#### 形容词
- 必要的、紧迫的、重要的
- 可行的、适宜的、妥当的
- 稳妥的、审慎的、严谨的
- 充分的、全面的、系统的

#### 连接词
- 为此、因此、鉴于
- 一方面、另一方面
- 首先、其次、再次
- 综上所述、总的来看

### 句式特点

#### 陈述句为主
```
✅ 拟采购办公设备15台
❌ 我们想买15台电脑
```

#### 被动句适当使用
```
✅ 已完成市场调研
✅ 经论证具备可行性
```

#### 长句与短句结合
- 说明性内容用长句，体现逻辑性
- 结论性内容用短句，突出重点

## 五、常见错误示例

### ❌ 口语化
```
错误：我们部门想要买几台新电脑
正确：拟采购办公自动化设备若干台
```

### ❌ 表述含糊
```
错误：大概需要几十万块钱
正确：预计需资金XX万元
```

### ❌ 逻辑不清
```
错误：第一要买设备，第二要安排人，第三要培训
正确：一是采购设备；二是配备人员；三是开展培训
```

### ❌ 缺少论证
```
错误：我们需要买电脑，请批准
正确：为提升工作效率，现有设备已老化......
      经评估，采购新设备具备必要性......
```

## 六、不同类型请示的侧重点

### 采购类
- 强调必要性（现有设备老化/不足）
- 市场调研情况
- 性价比分析
- 预算合理性

### 制度类
- 制度修订背景
- 主要变化内容
- 预期效果
- 实施步骤

### 人事类
- 人员需求分析
- 岗位职责说明
- 任职要求
- 招聘/选拔方案

### 项目类
- 项目背景和意义
- 主要内容和目标
- 实施计划和步骤
- 资源需求和保障
- 风险评估

---

## 七、数字与标点规范

### 数字使用
- **统计数字**：使用阿拉伯数字
  - 15台、100万元、2025年
- **约数**：中文表达
  - 数十个、数百万

### 标点符号
- **标题**：不用标点（除书名号）
- **序号**：一、、（一）、1、、（1）
- **并列**：用分号"；"
- **结尾**：用句号"。"（请示事项除外）

---

**遵循本指南，确保请示文档的专业性和规范性。**
